PraxiApp Backend (Development)

Du siehst gerade den DEV-Index. Viele Seiten sind geschützt und erfordern Login (Session oder JWT).

Tipp: Für „alle Seiten“ brauchst du i.d.R. einen Staff/Superuser (für Dashboard/Admin) und für API-Endpunkte zusätzlich eine Rolle (RBAC, z.B. admin).

Admins & Dashboards

  • /admin/ (Django Admin)
  • /praxiadmin/ (PraxiApp Admin)
  • /praxiadmin/dashboard/ (Haupt-Dashboard)
  • /praxiadmin/dashboard/scheduling/
  • /praxiadmin/dashboard/patients/
  • /praxiadmin/dashboard/doctors/
  • /praxiadmin/dashboard/operations/

API (GET/POST)

  • /api/health/ (öffentlich)
  • /api/auth/login/ (JWT Login, POST)
  • /api/auth/me/ (auth erforderlich)
  • /api/appointments/
  • /api/operations/
  • /api/op-dashboard/
  • /api/op-timeline/
  • /api/patients/

In DEV ist die Browsable API aktiv, aber Standard-Permission ist IsAuthenticated.

Was du wahrscheinlich als Nächstes willst

  • Als Admin einloggen, dann Dashboard öffnen.
  • Optional: Seed-Daten laden (damit Tabellen/Charts nicht leer sind).
